## 第 2 部分：商業目標 (Business Objectives) - 「為何做？」

*在投入資源前，我們必須清晰地定義問題、目標使用者和商業價值，避免開發出無人需要的產品。*

| 區塊 | 內容 |
| :--- | :--- |
| **1. 背景與痛點** | **核心問題**：現有人格測評工具缺乏可執行的決策建議<br>**痛點分析**：<br>• 市場上的人格測驗多停留在「標籤化」層面<br>• 缺乏從測驗結果到實際行動的轉換<br>• 管理者和教練難以基於測驗結果提供具體指導<br>• 職涯建議通常過於籠統，缺乏個人化<br>**影響範圍**：HR 專業人員、企業教練、個人發展諮詢師 |
| **2. 策略契合度** | **與整體目標的契合**：<br>• 建立基於心理測量學的決策支援系統<br>• 將公領域研究成果轉化為實用工具<br>• 探索 AI 輔助的個人化建議系統<br>• 為後續企業級應用奠定技術基礎 |
| **3. 成功指標 (Success Metrics)** | **主要指標**: 4週內完成端到端 MVP，90% 用戶能在 ≤5分鐘內完成測驗並獲取報告<br>**次要指標**: 管理者/教練滿意度 ≥4.5/5，建議採納率 ≥70% (7日內)<br>**技術指標**: 系統錯誤率 <1%，PDF生成 <1秒，所有建議可回溯到分數權重 |

| **US-005** | **As a** 測驗使用者,<br>**I want to** 下載個人化PDF報告,<br>**so that** 我能保存結果並與教練分享。 | 1. PDF包含完整分析結果<br>2. 專業格式易於分享<br>3. 生成時間 <1秒<br>4. 支援一次性分享連結 | `GET /report/{session}.pdf`<br>PDF生成引擎 |

## 第 4 部分：範圍與限制 (Scope & Constraints)

*明確定義邊界是專案成功的關鍵。這為 LLM 生成程式碼提供了最重要的「護欄」。*

| 區塊 | 內容 |
| :--- | :--- |
| **功能性需求 (In Scope)** | **核心流程**：<br>• 隱私同意和資料治理<br>• Mini-IPIP (20題) 測驗引擎<br>• Big Five/HEXACO 計分演算法<br>• 12個優勢面向映射系統<br>• 職缺推薦規則引擎<br>• 改善建議系統<br>• PDF 報告生成<br>• 一次性分享機制<br>**管理功能**：<br>• 審計軌跡和資料追蹤<br>• 可解釋性和 Provenance |
| **非功能性需求 (NFRs)** | **效能**: 測驗完成 ≤5分鐘，PDF生成 <1秒<br>**安全性**: 資料最小化、同意記錄、限時分享<br>**可靠性**: 錯誤率 <1%，90% 可用性<br>**可解釋性**: 所有建議可回溯到權重和規則<br>**相容性**: SQLite WAL模式，支援並發 |
| **不做什麼 (Out of Scope)** | **V1 不包含**：<br>• 複製 Gallup 官方測驗內容或商標<br>• 大規模團隊儀表板和管理系統<br>• 第三方系統整合 (HR系統、LDAP等)<br>• 多語言支援 (僅繁體中文)<br>• 行動 App (僅響應式網頁)<br>• 即時聊天或客服功能 |
| **假設與依賴** | **假設**: <br>• 使用者具備基本網路連線<br>• 使用者理解繁體中文<br>• Mini-IPIP 量表足以提供有效測量<br>**技術依賴**: <br>• Python 3.11+ 環境<br>• SQLite 資料庫<br>• FastAPI 框架<br>• ReportLab PDF 生成 |
