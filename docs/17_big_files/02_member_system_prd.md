## 第 2 部分:商業目標 (Business Objectives) - 「為何做?」

*在投入資源前，我們必須清晰地定義問題、目標使用者和商業價值，避免開發出無人需要的產品。*

| 區塊 | 內容 |
| :--- | :--- |
| **1. 背景與痛點** | **當前系統問題:**<br><br>目前 Gallup 優勢測驗系統採用匿名評測模式，使用者完成測驗後獲得一次性分享連結。此模式存在以下痛點:<br><br>1. **無法追蹤個人成長**: 使用者無法查看歷史評測記錄，無法了解自己的才幹發展軌跡<br>2. **報告易遺失**: 一次性分享連結過期後，使用者永久失去評測結果<br>3. **無法比較**: 使用者重複測驗時，無法與過往結果進行對比分析<br>4. **企業客戶需求**: HR/教練希望追蹤團隊成員的評測歷史，建立組織才幹地圖<br>5. **付費功能限制**: 無會員機制，無法實現進階功能的訂閱制商業模式<br><br>**影響範圍**: 個人使用者、企業 HR、職涯教練、教育機構 |
| **2. 策略契合度** | **支持公司戰略目標:**<br><br>1. **從一次性工具到持續服務**: 轉型為會員訂閱制 SaaS 平台<br>2. **數據資產積累**: 建立使用者才幹數據庫，支持未來 AI 推薦引擎<br>3. **企業客戶拓展**: 提供團隊管理功能，進入 B2B 市場<br>4. **提升使用者黏著度**: 從單次使用提升至長期關係經營<br><br>**契合 2025 Q4-Q1 戰略重點**: 建立會員基礎設施，為後續付費功能鋪路 |
| **3. 成功指標 (Success Metrics)** | **主要指標 (Primary KPIs)**:<br>- 會員註冊率: 完成評測的使用者中，≥40% 註冊為會員<br>- 會員留存率: 30 天留存 ≥60%<br>- 重複測驗率: 會員重測率 ≥15% (vs 匿名 <5%)<br><br>**次要指標 (Secondary KPIs)**:<br>- 歷史報告查閱次數: 平均每會員每月 ≥2 次<br>- 註冊轉化時間: 從完成評測到註冊 ≤5 分鐘<br>- 系統可用性: ≥99.5% uptime |

## 第 4 部分:範圍與限制 (Scope & Constraints)

*明確定義邊界是專案成功的關鍵。這為 LLM 生成程式碼提供了最重要的「護欄」。*

| 區塊 | 內容 |
| :--- | :--- |
| **功能性需求 (In Scope)** | **MVP 階段 (6 週開發):**<br>- 會員註冊/登入 (Email + 社交帳號)<br>- 密碼找回機制<br>- 個人檔案管理 (基本資料 + 隱私設定)<br>- 評測歷史列表 (時間軸檢視)<br>- 歷史報告下載 (PDF)<br>- 報告分享連結生成 (有時效性)<br>- 個人儀表板 (才幹摘要 + 評測統計)<br>- 會員權限管理 (基礎 RBAC)<br><br>**Post-MVP (未來迭代):**<br>- 評測結果對比分析<br>- 團隊管理功能 (B2B)<br>- 訂閱制付費功能<br>- 進階數據分析儀表板 |
| **非功能性需求 (NFRs)** | **性能:**<br>- API P95 延遲 <200ms<br>- 頁面首次載入 <2 秒<br>- 支援並發 500 使用者<br><br>**安全性:**<br>- 密碼使用 bcrypt 加密 (cost factor ≥12)<br>- JWT Token 有效期 7 天, Refresh Token 30 天<br>- 所有 API 強制 HTTPS (TLS 1.3+)<br>- 符合 OWASP Top 10 防護<br><br>**可用性:**<br>- 系統可用性 ≥99.5% (允許每月 3.6 小時維護)<br>- 資料備份每日自動執行<br>- RPO (資料遺失容忍) <24 小時<br>- RTO (恢復時間目標) <4 小時<br><br>**合規性:**<br>- 符合 GDPR 個人資料保護規範<br>- 提供完整資料匯出/刪除功能<br>- 審計軌跡保留 1 年 |
| **不做什麼 (Out of Scope)** | **明確排除項目 (避免範圍蔓延):**<br>- ❌ 不支援手機號碼註冊 (僅 Email + 社交帳號)<br>- ❌ 不支援多語系 (MVP 僅繁體中文)<br>- ❌ 不支援企業 SSO (如 SAML, LDAP) - Post-MVP<br>- ❌ 不支援即時通訊/評論功能<br>- ❌ 不支援手機 APP (僅 RWD 響應式網頁)<br>- ❌ 不支援離線模式<br>- ❌ 不支援第三方整合 (如 Slack, Teams) - Post-MVP |
| **假設與依賴** | **關鍵假設:**<br>- 使用者擁有有效的 Email 信箱<br>- 使用者使用現代瀏覽器 (Chrome 90+, Safari 14+, Firefox 88+)<br>- 平均評測歷史記錄 ≤10 筆/會員<br><br>**外部依賴:**<br>- **Email 服務**: SendGrid API (交易郵件發送)<br>- **OAuth 提供商**: Google OAuth 2.0, Facebook Login<br>- **儲存服務**: 現有 SQLite/檔案存儲系統<br>- **PDF 生成**: 現有報告生成引擎<br><br>**內部依賴:**<br>- 整合現有評測引擎 (V4.0 Thurstonian IRT)<br>- 整合現有報告系統<br>- 整合現有同意管理系統 |

## 📋 附錄:使用者旅程地圖 (User Journey Map)

### 新使用者完整旅程

```
1. 首次訪問網站
   ↓
2. 瀏覽著陸頁,了解服務價值
   ↓
3. 點擊「開始評測」
   ↓
4. 閱讀並同意隱私政策
   ↓
5. 完成 30 題評測 (~15 分鐘)
   ↓
6. 查看即時結果摘要
   ↓
7. 【決策點】是否註冊會員?
   ├─ 是 → 8a. 快速註冊 (Email/社交)
   │         ↓
   │        9a. 驗證 Email
   │         ↓
   │        10a. 評測結果自動綁定至帳號
   │         ↓
   │        11a. 導向個人儀表板
   │
   └─ 否 → 8b. 獲得一次性分享連結
           ↓
          9b. 下載 PDF 報告
          ↓
          10b. 離開網站 (可能遺失結果)
```

### 會員重複評測旅程

```
1. 登入會員帳號
   ↓
2. 個人儀表板查看歷史摘要
   ↓
3. 點擊「開始新評測」
   ↓
4. 完成評測
   ↓
5. 系統自動與歷史記錄比較
   ↓
6. 查看「才幹變化趨勢」報告
   ↓
7. 下載最新 PDF 報告
   ↓
8. 返回儀表板
```
